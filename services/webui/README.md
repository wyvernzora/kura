# kura web UI

Production web UI for kura. SPA built with React 18 + TypeScript + Vite +
Tailwind v4 + TanStack Router/Query + Storybook. Bundle is embedded into
the Go binary via `internal/server/webui` and served by the same kura
HTTP server that exposes `/api/v1/*`.

## Layout

```
web/
  package.json
  vite.config.ts                  # /api proxy → :8080, TanStack Router plugin, Tailwind v4
  tsconfig.{,app,node}.json
  biome.json                      # lint + format
  components.json                 # shadcn config
  index.html                      # Vite SPA entry
  .storybook/{main,preview}.ts
  src/
    main.tsx                      # React entry: QueryClientProvider + RouterProvider
    routeTree.gen.ts              # auto-generated by TanStack Router plugin (gitignored)
    routes/
      __root.tsx                  # root layout + <Outlet />
      index.tsx                   # P0 placeholder; P3 swaps in the design-system shell
    components/
      HelloKura.tsx               # placeholder content
      HelloKura.stories.tsx       # one Storybook story so Storybook boots green
    api/
      client.ts                   # fetch wrapper stub; PR 3 adds auth-mode awareness
    styles/
      globals.css                 # Tailwind base + minimal @theme tokens
```

## Dev workflow

Two paths — pick whichever matches how you're hacking.

**Inside the devserver container** (recommended): `make devserver-run`
from the repo root boots kura (`:8080`) + Vite (`:5173`) + Storybook
(`:6006`) in one container with hot reload across all three. Set
`KURA_WEB_DISABLED=1` for backend-only sessions.

**Host-only**:

1. Start kura on host: `KURA_DISABLE_TOKEN=1 KURA_LIBRARY_ROOT=... KURA_INBOX_ROOT=... go run ./cmd/kura serve --rest=127.0.0.1:8080`.
2. From `web/`: `pnpm install` once, then `pnpm dev` (Vite on `:5173`).
3. Optional: `pnpm storybook` for the component sandbox on `:6006`.

The Vite proxy forwards `/api/*` to `:8080`, so the browser sees same-origin
requests and CORS / bearer flows unchanged.

## Build

```
pnpm install        # one-time
pnpm build          # → web/dist/
```

The repo-level `make web-build` runs this and rsyncs the output to
`internal/server/webui/dist/` so the next `go build` embeds the new bundle.
`make build` does **not** auto-build the web bundle — keeps Go iteration fast.
`make release` runs both halves end to end.

## TanStack Router file-based routing

Routes live under `src/routes/`. The `@tanstack/router-plugin` Vite plugin
watches the directory and regenerates `src/routeTree.gen.ts` on every
change. The generated file is gitignored; do not edit it by hand.

Add a route by creating `src/routes/<path>.tsx` (e.g. `series.$ref.tsx` for
a `/series/$ref` route) with a `createFileRoute` export.

## Code style

- Biome handles lint + format. Run `pnpm lint:fix` before committing.
- Tailwind v4 — no separate `tailwind.config.js`; tokens live in
  `src/styles/globals.css` under `@theme`.
- Component imports use the `@/` alias mapped to `src/` (mirrored in
  `tsconfig.app.json`, `vite.config.ts`, and `.storybook/main.ts`).
